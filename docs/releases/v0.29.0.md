# Guarded Continuation Checker v0.29.0

This is the first GCC release with a frozen firmware and RTL support boundary.
It remains an evaluation-ready research release, not a production-qualified or
certified verification product.

## Release highlights

- Adds the `firmware-rtl-v1` build profile, containing exactly eight documented
  firmware and RTL commands and rejecting research commands before dispatch.
- Adds a deterministic static Linux candidate with SPDX SBOM, in-toto
  provenance, offline verification, tamper rejection and two-build byte
  reproducibility.
- Preserves firmware CLI v2 and artifact schema v4 for SAFE, replayable UNSAFE
  and operational-failure results.
- Adds proof-carrying OpenTitan semantic-revision experiments and a canonical
  shared-section revision batch format, while retaining the negative baseline
  showing that maintained Yosys, rIC3 and Certifaiger evidence is substantially
  smaller for the measured workload.
- Records an exact QatQ compression result that beats the strongest tested
  Zstandard configuration by 34.6 percent on the revision batch. QatQ remains
  outside the supported profile pending resource, portability and hostile-input
  qualification.

## Installation

Install the frozen firmware profile from crates.io:

```sh
cargo install guarded-continuation-checker --version 0.29.0 --locked \
  --features production-firmware
guarded-continuation-checker production-profile-version
```

The expected capability line is:

```text
production_support_profile=firmware-rtl-v1 firmware_cli_version=2 artifact_schema_version=4
```

Linux evaluators should prefer the attached static archive and verify its
checksum plus GitHub build and SBOM attestations before execution.

## Claim boundary

The release proves its own build, artifact and support-profile invariants. It
does not prove that an entire device is safe. Independent security assessment,
independent technical review and the documented multi-partner pilot cohort are
still required before GCC can be described as production-grade.
