# OpenTitan prim_count revision impact v1

## Status

Accepted local public-product mechanism evidence. The complete four-transition
answer gate passes. The maintained same-scope baseline, hosted portability, and
independent review remain open, so this is not a production or novelty claim.

## Question

Can one deterministic revision-impact certificate independently preserve all
four old-to-new bounded answer classes across an authentic public RTL semantic
revision, without per-formula calibration or fallback?

## Public revision and properties

The component sources are generated by pinned Yosys commit
`b8e7da6f40ae8f552c116bf6c359b07c6533e159` from OpenTitan `prim_count`
commit `369cffc85db0e6d5a667676a6f89987b94210e70` and its parent
`34157c7afb84a7be7b1b1250d673f9fa8a3c18ce`. The specialised sources and
their upstream-equivalence evidence are unchanged from the earlier semantic
revision and distinct-property query-service cohorts.

Four horizon-zero queries form the smallest retained set that covers every
answer transition:

| Query | Old | New | Class |
|---|---:|---:|---|
| unchanged environment `observed_bad` input | UNSAFE | UNSAFE | unchanged UNSAFE |
| component property 1000, `count == 0` | UNSAFE | SAFE | UNSAFE to SAFE |
| component property 1001, `count == 1` | SAFE | SAFE | unchanged SAFE |
| component property 1003, `count == 3` | SAFE | UNSAFE | SAFE to UNSAFE |

The environment query deliberately classifies `observed_full` as an external
input and checks the identical environment source in both masks. It is not
presented as a changed `prim_count` property. Its role is to prove that the
aggregate preserves an unchanged UNSAFE obligation alongside three distinct
public component properties.

## Exact result

The bundle has one changed atom, four queries, two counterfactual combinations,
and eight independently verified revision-local evidence members. It is
56,632,691 bytes with SHA-256
`8880f1ac6b8f62d8198b73d96adbf99469823358a104049b70564dd41b5dec76`.
Two clean productions were byte-identical and emitted identical logical result
and work fields. Independent verification reports:

- 56,632,176 parsed evidence bytes;
- eight semantic replays;
- 16 component validations;
- 1,310,720 composed pair checks;
- 16 final transition checks; and
- eight result comparisons.

No route or fallback is available for this command. Every query transition is
reported through the strict `transition-v1` response schema and parsed by the
typed `RevisionImpactTool` client.

## Claim boundary

This closes the four-transition public-product mechanism gate. It does not show
that exhaustive bounded counterfactuals, proof reuse, change impact, or minimal
invalidating sets are novel individually. It also does not show a performance
or artifact-size advantage. A pinned proof-producing maintained-tool baseline
at the same four-query scope must be measured before any narrower distinction
can advance.
