# OpenTitan prim_count revision impact v1

## Status

Accepted local public-product mechanism evidence with a disconfirming
maintained baseline. The complete four-transition answer gate passes. Hosted
format portability passes on a frozen certificate fixture. A larger multi-atom
public subsystem and independent review remain open, so this is not a
production or novelty claim.

## Question

Can one deterministic revision-impact certificate independently preserve all
four old-to-new bounded answer classes across an authentic public RTL semantic
revision, without per-formula calibration or fallback?

## Public revision and properties

The component sources are generated by pinned Yosys commit
`b8e7da6f40ae8f552c116bf6c359b07c6533e159` from OpenTitan `prim_count`
commit `369cffc85db0e6d5a667676a6f89987b94210e70` and its parent
`34157c7afb84a7be7b1b1250d673f9fa8a3c18ce`. The specialised sources and
their upstream-equivalence evidence are unchanged from the earlier semantic
revision and distinct-property query-service cohorts.

Four horizon-zero queries form the smallest retained set that covers every
answer transition:

| Query | Old | New | Class |
|---|---:|---:|---|
| unchanged environment `observed_bad` input | UNSAFE | UNSAFE | unchanged UNSAFE |
| component property 1000, `count == 0` | UNSAFE | SAFE | UNSAFE to SAFE |
| component property 1001, `count == 1` | SAFE | SAFE | unchanged SAFE |
| component property 1003, `count == 3` | SAFE | UNSAFE | SAFE to UNSAFE |

The environment query deliberately classifies `observed_full` as an external
input and checks the identical environment source in both masks. It is not
presented as a changed `prim_count` property. Its role is to prove that the
aggregate preserves an unchanged UNSAFE obligation alongside three distinct
public component properties.

## Exact result

The bundle has one changed atom, four queries, two counterfactual combinations,
and eight independently verified revision-local evidence members. It is
56,632,691 bytes with SHA-256
`8880f1ac6b8f62d8198b73d96adbf99469823358a104049b70564dd41b5dec76`.
Two clean productions were byte-identical and emitted identical logical result
and work fields. Independent verification reports:

- 56,632,176 parsed evidence bytes;
- eight semantic replays;
- 16 component validations;
- 1,310,720 composed pair checks;
- 16 final transition checks; and
- eight result comparisons.

No route or fallback is available for this command. Every query transition is
reported through the strict `transition-semantic-set-v1` response schema and
parsed by the
typed `RevisionImpactTool` client.

## Maintained-tool baseline

Pinned Yosys emits eight same-scope AIGER scenario models. Qualified rIC3
produces four SAFE witnesses and four UNSAFE traces. Qualified Certifaiger
checks every witness or trace independently. All answers and all four
transition classes agree with GCC.

The eight selected models total 1,748 bytes and their evidence totals 910 bytes. The
complete maintained package is therefore 2,658 bytes. GCC's 56,632,691-byte
aggregate is about 21,306.5 times larger. GCC's source-through-production path
peaks at 324,780,032 bytes and independent checking at 270,581,760 bytes. The
maintained route's source-through-production path peaks at 18,317,312 bytes,
and its largest Certifaiger checker job at 4,722,688 bytes. The current GCC
peaks are therefore about 17.7 and 57.3 times larger.

Pinned Yosys synthesis plus GCC certificate production takes 3.03 seconds. The
same exact-scope maintained route takes 1.81 seconds, making GCC about 1.67
times slower on this small cohort. GCC verification takes 0.79 seconds and the
maintained evidence checks take 1.943 seconds, but the maintained observation
includes eight separate network-disabled Docker launches. The checker process
boundaries differ, so the apparent GCC checker advantage is not used for a
speed claim.

## Claim boundary

This closes the four-transition public-product gate and the local maintained
semantic, transfer-size, evidence-checking, source-to-answer timing, and
peak-RSS baseline slices. It
does not show
that exhaustive bounded counterfactuals, proof reuse, change impact, or minimal
invalidating sets are novel individually. It also does not show a performance
or artifact-size advantage. The baseline strongly falsifies artifact
efficiency for the current duplicated-evidence container. A larger multi-atom
public revision with nontrivial minimal invalidating sets, shared-section
storage, larger public-subsystem evidence, and independent review remain
necessary before any narrower distinction can advance.
